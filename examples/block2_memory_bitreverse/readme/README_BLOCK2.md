# Block 2 – Memory and Bit-Reversal Reordering

## General Description

Block 2 is responsible for receiving complex samples from Block 1, temporarily storing them in memory, and subsequently delivering them in **bit-reversed order** to Block 4 (FFT Controller).

This block implements the permutation stage required by a Radix-2 DIT (Decimation-In-Time) FFT, where input data must be reordered before butterfly processing.

---

# Block Architecture

The block is composed of the following modules:

```text
block2_memory_bitreverse_top
│
├── permutation_controller
│
├── bit_reverse
│
└── dual_port_ram_buffer
```

---

# Top-Level Module

## block2_memory_bitreverse_top.v

This module represents the official interface of Block 2 and encapsulates all internal functionality.

### Functions

* Receive complex samples from Block 1.
* Manage temporary storage using RAM.
* Reorder data using bit-reversed addressing.
* Deliver reordered data to Block 4 through a valid/ready handshake protocol.

---

# bit_reverse Module

## File

```text
rtl/bit_reverse.v
```

## Function

Computes the bit-reversed address corresponding to a given input index.

### Example

For N = 8:

```text
0 → 0
1 → 4
2 → 2
3 → 6
4 → 1
5 → 5
6 → 3
7 → 7
```

For N = 1024:

```text
1 → 512
2 → 256
3 → 768
```

### Inputs

| Signal   | Description    |
| -------- | -------------- |
| index_in | Original index |

### Outputs

| Signal    | Description        |
| --------- | ------------------ |
| index_out | Bit-reversed index |

# dual_port_ram_buffer Module

## File

```text
rtl/dual_port_ram_buffer.v
```

## Function

Dual-port memory used to store complex samples.

### Features

* Synchronous write operation.
* Registered synchronous read operation.
* Separate memory arrays for real and imaginary components.
* Storage capacity of 1024 complex samples.

### Inputs

| Signal  | Description                  |
| ------- | ---------------------------- |
| clk     | System clock                 |
| wr_en   | Write enable                 |
| wr_addr | Write address                |
| wr_real | Real component to store      |
| wr_imag | Imaginary component to store |
| rd_addr | Read address                 |

### Outputs

| Signal  | Description                   |
| ------- | ----------------------------- |
| rd_real | Retrieved real component      |
| rd_imag | Retrieved imaginary component |

# permutation_controller Module

## File

```text
rtl/permutation_controller.v
```

## Function

Implements the finite state machine (FSM) that controls the complete operation of Block 2.

### Responsibilities

* Control data writes into RAM.
* Generate bit-reversed read addresses.
* Coordinate memory read operations.
* Manage the valid/ready handshake toward Block 4.

### FSM States

```text
IDLE
WRITE
SET_READ
WAIT_READ
OUTPUT_VALID
DONE
```

### State Description

#### IDLE

Waits for the beginning of a new frame using the signal:

```text
frame_start
```

#### WRITE

Stores 1024 complex samples in natural order.

#### SET_READ

Computes the bit-reversed address corresponding to the current index.

#### WAIT_READ

Waits for the RAM read latency.

#### OUTPUT_VALID

Presents a valid output sample to Block 4.

#### DONE

Completes the current frame and returns to the IDLE state.

# Handshake Protocol

Block 2 uses a standard valid/ready handshake protocol.

## Input

```text
br_ready
```

Generated by Block 4.

## Output

```text
br_valid
```

Generated by Block 2.

A transfer occurs only when:

```text
br_valid = 1
br_ready = 1
```

# Expected Behavior

Block 2 receives complex samples in natural order:

```text
0, 1, 2, 3, 4, 5, 6, 7
```

and outputs them in bit-reversed order:

```text
0, 4, 2, 6, 1, 5, 3, 7
```

while preserving the original real and imaginary sample values.
